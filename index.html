Basics

And how they work

below I will show you a few basics and why we need them

Tags

The content of a web page is marked up using HTML tags. For example, "html" "body" "p" and more. Tags are building blocks of a website. they can be named classes and changed with css. There are also semantics tags but it's not important. If you want to be a good Front-end developer you need to know them link to picture

Templates

No one likes to do the same thing again and again so a lot of people make their templates for faster work. And I also make my own templates and after I just copy the right pieces and I dont need to write the same things all over again. In my opinion this is a very useful method, isn't it?

Js - do we need it?

JavaScript is a programming language which we can use with HTML and css. In a lot of cases we dont need to use it. But sometimes we can write only 5 lines in JS or 100 and more line in css? I think everyone will chose the first option. Js is a very difficult language but luckily we dont need to know all the language - only a small piece for web-developing. Also we have a plugin JQuery which helps with writing js code.

How many people need to work?

In the whole procces a minimum of 3 people work: designer, front-End developer and Back-end Developer In the next slides we will see the difference between them and other important things

A person who creates a layout according to the words of a client - One of the most difficult stages in the cycle of creating a website. If the client wants to change somethimg again and again and again you will get angry because in the end you change a half of your work and you waste your time because "the client wanted it that way"

Graphic Designer

Front-end developer is a person who works after designer so you need to know how to change or export something from the layout and add it to website. Usually designer doesnt make a layout version for phones and you need to imagine how it needs to look like and to be useful for user

Front-end developer

Back-End developer is a person who works with the server part of the site. Programming language: PHP. This is the most difficult stage of making a website after the design. Front-end developer needs to understand what will happen next with his HyperText Markup and think ahead, so that Back-end developer can work easier with your code (if you are a Front-end developer)
